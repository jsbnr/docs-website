[New Relic のシンセティックモニタリング](/docs/synthetics/new-relic-synthetics/getting-started/introduction-new-relic-synthetics) レポートされる [イベントデータ](/docs/using-new-relic/data/understand-data/new-relic-data-types#event-data) 一部の UI ディスプレイに表示され、 [クエリやチャートの作成にも利用できます](/docs/using-new-relic/data/understand-data/query-new-relic-data) 。以下の表でイベント名を選択すると、その属性が表示されます。

<table>
  <thead>
    <tr>
      <th style={{ width: "300px" }}>
        イベント
      </th>

      <th>
        説明
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td id="syntheticcheck-error">
        [`SyntheticCheck`](/attribute-dictionary/?event=SyntheticCheck)
      </td>

      <td>
        `SyntheticCheck`は、特定のモニターを1回実行して生じるメトリクスを返します。このメトリクスには、モニターの処理時間情報、モニターチェックのロケーション、リクエストのサイズ、レスポンスヘッダー、モニターのタイプ、タイムスタンプが含まれます。

        合成モニターがチェックするたびに、チェックに関する詳細が`SyntheticCheck`イベントタイプで取得されます。`SyntheticCheck`イベントには、ステータス、モニターのタイプ、リクエストのサイズ、レスポンスヘッダーなど、チェック固有の詳細情報が含まれ、可視性を提供します。
      </td>
    </tr>

    <tr>
      <td id="syntheticrequest">
        [`SyntheticRequest`](/attribute-dictionary/?event=SyntheticRequest)
      </td>

      <td>
        `SyntheticRequest`は、チェックの間に行われた各HTTPリクエストからの結果を返します。収集されたデータには、ジョブ情報、ロケーション、リクエストの内容のタイプ、処理時間情報、リクエストのサイズ、ページロード情報が含まれます。

        シンプルモニターチェックまたはスクリプト化モニターチェックのたびに、チェック中に行われた各HTTPリクエストをキャプチャします。HTTPの詳細情報は、`SyntheticsCheck`イベントタイプよりも詳しいレベルでキャプチャされます。
      </td>
    </tr>

    <tr>
      <td>
        [`SyntheticPrivateLocationStatus`](/attribute-dictionary/?event=SyntheticsPrivateLocationStatus)
      </td>

      <td>
        プライベートロケーションで実行されるすべてのモニターチェックにより、そのプライベートな場所の処理能力の詳細がトリガされます。これらの詳細は、`SyntheticPrivateLocationStatus`イベントで取得されます。これにより、プライベートロケーションの能力や、ワークロードのサポートに追加のミニオンが必要かどうかが可視化されます。
      </td>
    </tr>

    <tr>
      <td id="synthetic-private-minion">
        [`SyntheticPrivateMinion`](/attribute-dictionary/?event=SyntheticsPrivateMinion)
      </td>

      <td>
        ファイアウォールの内側のロケーションなどのプライベートロケーションがある場合は、`SyntheticsPrivateMinion`イベントを使用してこれらのロケーションに関する情報を表示できます。

        各プライベートミニオンが実行するたびに、30秒ごとにヘルス詳細が`SyntheticPrivateMinion`に送信されます。これにより、そのロケーションで実行するプライベートミニオンのヘルス状態を理解できます。
      </td>
    </tr>
  </tbody>
</table>
