[New Relic REST API](/docs/apis/rest-api-v2/getting-started/introduction-new-relic-rest-api-v2) を使用する際には、いくつかの ID プレースホルダー ( `$APP_ID`, `$KEYTX_ID`, `$COMPONENT_ID` など) をアプリ、トランザクション、またはその他のコンポーネントの ID に置き換える必要があります。

[New Relic One](/docs/new-relic-one/use-new-relic-one/get-started/introduction-new-relic-one) を使用して、これらの ID の一部を取得することができます。また、 [REST API Explorer](/docs/apis/rest-api-v2/get-started/list-application-id-host-id-instance-id) を使って、ID の一覧を取得することもできます。

## New Relic Oneを使ったIDの検索 [#ui]

New Relic One では、アプリケーションやその他一部のコンポーネントの ID をメタデータパネルに表示しています。

<img
  title="Metadata panel in NR1"
  alt="Metadata panel in NR1"
  src={img0MetadataPanelInNr1}
/>

<figcaption>
  アプリケーション名の横にある <Icon name="fe-info"/>アイコンをクリックすると、メタデータパネルが表示されます。
</figcaption>

以下にいくつか例を示します。

<CollapserGroup>
  <Collapser
    id="apm"
    title="APMアプリID"
  >
    `$APP_ID` をNew Relic UIから探す。

    1. [one.newrelic.com](https://one.newrelic.com) から、 **APM** をクリックし、アプリケーションを選択します。

    2. アプリケーション名の横にある <Icon name="fe-tag"/>アイコンをクリックすると、メタデータパネルが表示されます。

    3. **App ID** を探してください。

       REST API の呼び出しでは、この数字を `$APP_ID` に置き換えてください。
  </Collapser>

  <Collapser
    id="browser"
    title="ブラウザアプリID"
  >
    `$APP_ID` をNew Relic UIから探す。

    1. [one.newrelic.com](https://one.newrelic.com) から、 **ブラウザ** をクリックし、監視対象のアプリを選択します。

    2. アプリケーション名の横にある <Icon name="fe-tag"/>アイコンをクリックすると、メタデータパネルが表示されます。

    3. **App ID** を探してください。

       REST API の呼び出しでは、この数字を `$APP_ID` に置き換えてください。
  </Collapser>

  <Collapser
    id="mobile"
    title="モバイルアプリID"
  >
    `$MOBILE_ID` をNew Relic UIから探す。

    1. [one.newrelic.com](https://one.newrelic.com) から、 **Mobile** をクリックし、監視対象のアプリを選択します。

    2. アプリケーション名の横にある <Icon name="fe-tag"/>アイコンをクリックすると、メタデータパネルが表示されます。

    3. **App ID** を探してください。

       REST APIの呼び出しでは、この数字を `$MOBILE_ID` に置き換えてください。
  </Collapser>
</CollapserGroup>
