OpenTelemetry を New Relic One で実装するためのガイダンスが必要な場合、このリファレンスアーキテクチャは、ソフトウェア開発者、DevOps、エンタープライズアーキテクト、テクノロジーマネージャがプロジェクトで協働するために役立つはずです。これは、New Relic で OpenTelemetry を実装する一般的な方法を俯瞰的に示したテンプレートです。

この図は、OpenTelemetry、Prometheus、Jaeger、New Relic APMエージェントからテレメトリーを取り込む実装を表しています。3つのサービス（A、B、C）は、メトリクス、トレース、ログを送信するようにインスツルメンテーションされています。

OpenTelemetryで収集したデータについては、OTLPエンドポイント経由でNew Relicに直接送るか、オプションのOpenTelemetryコレクターにまず送るかのどちらかです。コレクターを使用する場合、テレメトリーがバックエンド (データ分析用の Kafka とエンドツーエンド観測用の New Relic One) に送信される前にデータパイプラインが集約され、強化されます。
