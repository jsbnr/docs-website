New Relic [Infrastructure integrations SDK](/docs/intro-infrastructure-integration-sdk) を使用してカスタムのオンホスト統合を作成する前に、お使いのシステムが以下の要件を満たしていることを確認してください。

## インフラ版 [#agent-version]

infrastructure Integrations SDK v4を使用するには、Infrastructure agentのバージョンが1.13.0以上である必要があります。

SDK v3を使用するには、エージェントのバージョンが1.0.888以上である必要があります。

インフラストラクチャーのエージェントのバージョン情報、インストールとアップデートのオプションについては、 [インフラストラクチャーのリリースノート](/docs/release-notes/infrastructure-release-notes/infrastructure-agent-release-notes) を参照してください。

SDKのバージョンや変更点については、 [チェンジログ](#change-log) をご覧ください。

## オペレーティングシステム [#os]

本SDKで構築されたインテグレーションは、LinuxまたはWindowsのどちらのOSでもコンパイル可能です。

## データとファイルの仕様 [#data-specs]

インフラストラクチャのオンホスト統合は、 [データおよびファイルの仕様](/docs/infrastructure/integrations-sdk/get-started/create-activate-infrastructure-integration) に準拠している限り、あらゆるプログラミング言語で作成することができます。
