## 問題

[New Relic C SDK](/docs/install-c-agent-compile-link-your-code) をインストールし、アプリのトラフィックを生成して数分待ちましたが、APM でデータのレポートが表示されませんでした。

## 解決策

これらのトラブルシューティングのヒントは、最も一般的な問題から最も一般的でない問題の順に並んでいます。

1. [C SDKとデーモンのログファイル](/docs/generate-logs-troubleshooting-c-sdk) の出力にエラーがないか確認してください。

2. [デーモン・プロセス（`newrelic-daemon` ）](/docs/install-c-sdk-compile-link-your-code#daemon) が起動していることを確認し、 計装化されたアプリケーションを起動する前に **を起動するようにします。**

   * * *

3. _\*\* アプリの名前とライセンスキーを [C SDK configuration](/docs/c-sdk-configuration) で確認してください。_ 必ず電話してください。

   * [`newrelic_create_app()`](https://newrelic.github.io/c-sdk/libnewrelic_8h.html#a61dd90439ae3cc5060021f6ab4701132)
   * [`newrelic_start_web_transaction()`](https://newrelic.github.io/c-sdk/libnewrelic_8h.html#adbf7c1fa57482f6e0a7f291e0b5ec80f) or [`newrelic_start_non_web_transaction()`](https://newrelic.github.io/c-sdk/libnewrelic_8h.html#a88648cc287f8d7e371139dc3809b7afb) as applicable
   * [`newrelic_end_transaction()`](https://newrelic.github.io/c-sdk/libnewrelic_8h.html#acbd88933d620c21b7b19f12b6f042da8)

     また、これらの呼び出しが `false` や `NULL` を返さないことを確認してください。\* 必要に応じて、 [`newrelic_version()`](https://newrelic.github.io/c-sdk/libnewrelic_8h.html#a8aec87e54af38e466fc5f715ef591d10) を使ってC SDKのバージョンを確認し、 [C SDKライブラリ](/docs/update-your-c-sdk-library) を最新のリリースに更新してください。\*\*

* * *
