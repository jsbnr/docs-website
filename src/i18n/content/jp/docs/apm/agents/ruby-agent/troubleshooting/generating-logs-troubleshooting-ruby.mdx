詳細なデバッグログは、標準的なRubyエージェントのインストールのトラブルシューティングに役立ちます。

<Callout variant="important">
  Herokuを使用している場合は、 [Ruby agent and Heroku](/docs/agents/ruby-agent/miscellaneous/ruby-agent-heroku#logging) を参照してください。
</Callout>

## ログファイルの生成 [#generating]

詳細なNew Relic for Rubyのログファイルを生成するため。

1. `newrelic.yml` ファイルをエディタで開きます。

2. [`log_level`](/docs/agents/ruby-agent/installation-configuration/ruby-agent-configuration#log_level) を `debug` に変更し、ファイルを保存します。インデントは変更しないでください。

   <Callout variant="important">
     `debug` は、非常に冗長なログレベルです。デバッグレベルのログを生成した後は、必ず以前の `log_level` に戻してください。
   </Callout>

3. Webアプリケーションを再起動します。

4. あなたのアプリにトラフィックを発生させ、5分ほど実行します。

5. ログファイルにエラーがないか確認します。デフォルトでは、ログファイルはアプリのlogフォルダにあり、名前は `newrelic_agent.log` となっています。

6. ログファイルをNew Relicサポートに送る場合は、ログファイルをサポートチケットに添付し、 `newrelic.yml` 、あなたのGemfile、 `Gemfile.lock` も一緒に添付してください。

7. `newrelic.yml` を再度編集し、 [`log_level`](/docs/agents/ruby-agent/installation-configuration/ruby-agent-configuration#log_level) を以前の設定に変更します。ファイルを保存します。
