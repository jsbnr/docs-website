## 問題

New Relic PHP エージェントは、スレッディングを伴う Apache のマルチプロセッシングモデル (MPM) をサポートしていません。

## 解決策

PHP 言語開発チーム [は、Apache](http://www.php.net/manual/en/install.unix.apache2.php "新しいウィンドウで開くリンク") でのスレッド MPM の使用を推奨していません。また、同様の安定性の理由から、PHP エージェントではスレッド MPM のサポートを実装しないことにしました。これには **worker** や **event** の MPM が含まれます。

デフォルトでスレッド化されていない **prefork** MPM がパフォーマンス上の理由でニーズに合わない場合は、代わりに **mod_fcgid** と **php-fpm** を使って FastCGI 経由で PHP を使用することをお勧めします。
