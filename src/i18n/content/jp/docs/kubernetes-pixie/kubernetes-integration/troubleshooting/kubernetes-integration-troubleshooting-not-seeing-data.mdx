## 問題

New Relic の Kubernetes 統合のための [のインストール手順](/docs/kubernetes-monitoring-integration#install) を完了しましたが、New Relic のアカウントに Kubernetes のデータが表示されません。

## 解決策

インフラストラクチャエージェント [を展開し、 ](/docs/infrastructure/install-infrastructure-agent/get-started/install-infrastructure-agent/#install)[Kubernetes のインストール手順](/docs/kubernetes-monitoring-integration#install) を完了したにもかかわらず、Kubernetes ダッシュボードに何の情報も表示されない場合は、以下の手順に従ってください。

1. この機能へのアクセス権があるかどうかを確認します。 [機能やデータへのアクセスに影響する要因](/docs/accounts/accounts-billing/general-account-settings/factors-affecting-access-features-data) をご参照ください。

2. 配置ファイルを実行する前に、 `CLUSTER_NAME` と `NRI_LICENSE_KEY` の変数が設定されていることを確認してください。詳細は、 [インストール手順](/docs/integrations/host-integrations/host-integrations-list/kubernetes-monitoring-integration#install) を参照してください。

3. を実行して、デプロイが成功したことを確認します。

   ```
   kubectl get pods --all-namespaces -o wide | grep newrelic
   ```

4. [インフラストラクチャエージェント](/docs/integrations/kubernetes-integration/troubleshooting/get-logs-version) からログを取得し、 [エラーメッセージがないことを確認する](/docs/integrations/host-integrations/troubleshooting/kubernetes-integration-troubleshooting-error-messages).
