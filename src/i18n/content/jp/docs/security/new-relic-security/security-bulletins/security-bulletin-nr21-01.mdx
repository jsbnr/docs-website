## 概要

ブラウザエージェントのセキュリティアップデートにより、 `file://` URIスキームを検出し、見つかった場合はそれ以降の実行とデータ収集を停止します。

**発売日：** 2021年3月9日

**脆弱性の識別子:** NR21-01

**Priority:** Medium

## 脆弱性情報 [#vuln-info]

ブラウザは、RFC 8089 で説明されている `file://` URI スキームを使用して、ホストマシン上のローカルファイルをレンダリングできます。エージェントのハーベストサイクルでは、この `file://` URI が pageURL データポイントとして記録されます。これにより、保存されたウェブページのディレクトリパスや、ディレクトリパスに含まれる名前や会社の情報など、ローカルファイルパスに含まれる機密性の高いデータが収集される可能性があります。 `file://` URIの詳細については、 [RFC 8089に記載されています。](https://tools.ietf.org/html/rfc8089)

### 緩和要因 [#factors]

ブラウザエージェントが設定されたウェブページをダウンロードし、そのファイルをブラウザで開く必要があります。 `file://` というURIスキームを使わずに読み込まれたHTMLファイルは影響を受けません。

## 回避策

* [最新のブラウザ監視エージェントに更新します。](/docs/browser/new-relic-browser/installation/update-browser-agent)
