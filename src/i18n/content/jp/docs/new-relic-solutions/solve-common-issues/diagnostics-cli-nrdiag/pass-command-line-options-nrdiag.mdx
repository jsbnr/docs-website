Diagnostics CLIで以下のコマンドラインオプションを使用するには、次のようにします。

<table>
  <thead>
    <tr>
      <th style={{ width: "250px" }}>
        オプション
      </th>

      <th>
        使用方法
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>
        `-アタッチメントエンドポイント STRING`
      </td>

      <td>
        サポートチケットに添付するアタッチメントのエンドポイントです。
      </td>
    </tr>

    <tr>
      <td>
        `-a`

        `-アタッチ`
      </td>

      <td>
        New Relic アカウントへの自動アップロードのために添付します。これには、お使いの環境で認証されたライセンスキーを使用します。
      </td>
    </tr>

    <tr>
      <td>
        **デプリケートされた**

        `-ストリング`

        `-アタッチメントキー STRING`
      </td>

      <td>
        サポートチケットに自動アップロードするための添付ファイルキーです。これは、既存のチケットからアタッチメントキーを取得します。(注：この機能はまもなく削除されます。上記の自動アップロードオプションをご利用ください。）
      </td>
    </tr>

    <tr>
      <td id="browser-check">
        `-ブラウザのURL STRING`
      </td>

      <td>
        Diagnostics CLIバージョン1.1.9以上

        これを起動すると、ブラウザモニタリングに関連する診断チェックのみが実行されます。このコマンドは、New Relic のブラウザ監視エージェントが存在するかどうかをチェックし、エージェントのバージョン、注入方法 (APM による[](/docs/browser/new-relic-browser/installation/install-new-relic-browser-agent#select-apm-app) またはコピー/ペーストによる [](/docs/browser/new-relic-browser/installation/install-new-relic-browser-agent#copy-paste-app))、ローダーのタイプ (Pro、Lite、SPA) を返します。イントラネットサイトのトラブルシューティングの際に、New Relic サポートに詳細を提供するために使用します。
      </td>
    </tr>

    <tr>
      <td>
        `-c STRING`

        `-設定ファイル STRING`
      </td>

      <td>
        デフォルトのエージェント設定ファイルの場所を上書きします。デフォルトのフォルダに加えて検索するフォルダ、または特定の設定ファイルへのパスを指定するために使用できます。
      </td>
    </tr>

    <tr>
      <td>
        `--フィルター STRING`
      </td>

      <td>
        タスクの結果を結果のステータスでフィルタリングします。カンマ区切りのリストを指定します。Accepts `Success`, `Warning`, `Failure`, `Error`, `None`, or `Info`.構文例。

        ```
        \"Success,Warning,Failure"
        ```
      </td>
    </tr>

    <tr>
      <td>
        `-h`

        `--ヘルプ`
      </td>

      <td>
        コマンドラインオプションの完全なリストを表示します。実行されるすべてのタスクをリストアップするには、 `-h tasks` を使用します。タイプ： bool
      </td>
    </tr>

    <tr>
      <td>
        `-inNewRelicCLI`
      </td>

      <td>
        タイプ: bool
      </td>
    </tr>

    <tr>
      <td>
        `-インタラクティヴ`
      </td>

      <td>
        タイプ: bool
      </td>
    </tr>

    <tr>
      <td>
        `-出力パス STRING`
      </td>

      <td>
        `nrdiag-output.zip` 、 `nrdiag-output.json` 、 `nrdiag-filelist.txt` の結果を書き込む別の出力ディレクトリを指定します。デフォルトの場所は `./` です。
      </td>
    </tr>

    <tr>
      <td>
        `-o`

        `--オーバーライド STRING`
      </td>

      <td>
        New Relic Support から要求されたときにオーバーライドするための引数を渡します。フォーマット： `identifier.property=value`.

        例の構文です。

        ```
        --override Java/Config/Agent.Status=Success
        ```

        ```
        -o Base/Config/Validate.agentlanguage=PHP
        ```
      </td>
    </tr>

    <tr>
      <td>
        `-p STRING`

        `--proxy STRING`
      </td>

      <td>
        HTTP接続タスクで使用されるプロキシを提供します。HTTP または HTTPS を指定します。プロキシは、 `http(s)://proxyIp:proxy` という形式でなければなりません。Diagnostics CLI がエージェントの設定ファイルでプロキシを見つけた場合、デフォルトでそのプロキシを使用します。ほとんどの場合、 `ポート` は必要ありません。Diagnostics CLI がエージェント・コンフィグ・ファイルでプロキシを見つけた場合、そのプロキシをデフォルトで使用します。
      </td>
    </tr>

    <tr>
      <td>
        `-proxy-pw STRING`
      </td>

      <td>
        必要に応じて、プロキシのパスワードを入力します。Diagnostics CLIがエージェント設定ファイルでプロキシを見つけた場合、デフォルトでそのプロキシを使用します。
      </td>
    </tr>

    <tr>
      <td>
        `-proxy-user STRING`
      </td>

      <td>
        必要に応じて、プロキシのユーザ名。Diagnostics CLIがエージェント設定ファイルでプロキシを見つけた場合、デフォルトでそのプロキシを使用します。
      </td>
    </tr>

    <tr>
      <td>
        `-q`

        `--静かに`
      </td>

      <td>
        Quiet出力は、高レベルの結果のみを表示し、説明的な出力は表示しません。 `-y` も使用されている場合、ファイル追加の警告を抑制します。 `-v` と矛盾しません。包含フィルタは無視されます。タイプ：bool
      </td>
    </tr>

    <tr>
      <td>
        `-qq`

        `--VeryQuiet`
      </td>

      <td>
        非常に静かな出力は、出力のための1つの要約行を印刷するだけです（ `q` を意味します）。 `-y` も使用されている場合、ファイル追加の警告を抑制します。 `-v` と矛盾しません。包含フィルタは無視されます。タイプ：bool
      </td>
    </tr>

    <tr>
      <td>
        `-SkipVersionCheck`
      </td>

      <td>
        アプリケーションの新しいバージョンの自動チェックをスキップする。タイプ：bool
      </td>
    </tr>

    <tr>
      <td>
        `-ShowOverrideHelp`
      </td>

      <td>
        タイプ: bool
      </td>
    </tr>

    <tr>
      <td>
        `-s STRING`

        `--スィーツ STRING`
      </td>

      <td>
        [スイートの実行](/docs/using-new-relic/cross-product-functions/diagnostics-cli-nrdiag/run-diagnostics-cli-nrdiag/#suites) 、特定の製品や課題を対象としたタスクの集まりです。複数のスイートを指定するには、カンマで区切ります。すべてのスイートのリストを取得するには、次を実行します。

        ```
        ./nrdiag -h suites
        ```
      </td>
    </tr>

    <tr>
      <td>
        `-t STRING`

        `--タスクはSTRING`
      </td>

      <td>
        エージェント別またはタスクタイプ別に、タスクのサブセットのみを実行します。複数のタスクを指定するには、カンマで区切るか、ワイルドカード `*` を使用します。すべてのタスクのリストを表示するには、次のように実行します。

        ```
        ./nrdiag -h tasks
        ```
      </td>
    </tr>

    <tr>
      <td>
        `-ユサージュ・オプトアウト`
      </td>

      <td>
        この実行について、匿名の Diagnostics CLI ツールの使用状況データを New Relic に送信することを辞退する。タイプ： bool
      </td>
    </tr>

    <tr>
      <td>
        `-v`

        `--verbose`
      </td>

      <td>
        チェック実行時に詳細なログを表示します。デフォルトでは無効です。タイプ：bool
      </td>
    </tr>

    <tr>
      <td>
        `--バージョン`
      </td>

      <td>
        現在の Diagnostics CLI のバージョンを表示します。これはまた、新しいバージョンをチェックするプロンプトを表示し、新しいバージョンが利用可能な場合はダウンロードするプロンプトを表示します。 `no-version-check` よりも優先されます。タイプ： bool
      </td>
    </tr>

    <tr>
      <td>
        `-y`

        `--はい`

        `--YesToAll`
      </td>

      <td>
        実行中に出てきたプロンプトに対して、 `yes` と応答します。デフォルトでは無効になっています。タイプ：bool
      </td>
    </tr>
  </tbody>
</table>
