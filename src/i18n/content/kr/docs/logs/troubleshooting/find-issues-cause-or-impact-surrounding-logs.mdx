## 문제

근본 원인을 식별할 수 있도록 문제가 발생한 원인을 확인하려고 합니다. 또는 이벤트의 영향을 평가할 수 있도록 이벤트 이후에 발생한 내용을 확인하려고 합니다. 그러나 관심 있는 로그 행을 쿼리하면 일치하지 않는 모든 행이 필터링됩니다.

## 해결책

쿼리한 로그 데이터 앞이나 뒤에 오는 행은 쿼리된 행의 컨텍스트를 이해하는 데 도움이 되는 경우가 많습니다. 기본적으로 New Relic은 쿼리를 둘러싼 30분의 로그를 표시합니다(선택한 로그 전 최대 15분 및 후 15분).

1. **[one.newrelic.com > 로그](https://one.newrelic.com)** 로 이동합니다.

2. 필요한 로그 정보 유형에 초점을 맞추는 쿼리를 입력한 다음 **쿼리 로그** 를 클릭합니다.

3. 자세히 살펴보고 싶은 로그 라인을 클릭하세요.

4. 속성 목록에서 **주변 로그 표시** 를 클릭합니다.

   <Icon name="fe-eye"/>

   상.

선택한(강조 표시된) 로그 행이 고정되어 주변 로그를 스크롤하는 동안 계속 볼 수 있습니다. 선택한 로그를 변경하려면 다음 중 하나를 수행합니다.

* **주변 로그 숨기기를** 클릭합니다.

  <Icon name="eye-off"/>

  상.

* 다른 로그 라인을 선택하십시오.

* 다른 쿼리를 실행합니다.
